## Data Model

```java
// Task fields
String id;          // UUID generated by server (UUID.randomUUID().toString())

String description; // optional, max 1000 chars
boolean completed;  // default false
String createdAt;   // ISO 8601 UTC (Instant.now().toString())
```

## Validation Rules

- `title` absent or blank в†’ 400: `{"error": "title is required"}`
- `title` > 200 chars в†’ 400: `{"error": "title must not exceed 200 characters"}`
- `description` > 1000 chars в†’ 400: `{"error": "description must not exceed 1000 characters"}`
- ID not found в†’ 404: `{"error": "Task not found"}`

**CRITICAL вЂ” ID path variable**: Use `@PathVariable String id` (NOT `@PathVariable UUID id`).
Then call `UUID.fromString(id)` in a try/catch вЂ” if `IllegalArgumentException`, throw `TaskNotFoundException` (в†’ 404).
Using `UUID` as path variable type causes Spring to return 400 for non-UUID strings, which breaks E2E-07/09/11.

---

## PUT Partial Update

Only update fields that are present in the request. If `title` is not in the request, keep the existing value. Always refresh `updatedAt`.
