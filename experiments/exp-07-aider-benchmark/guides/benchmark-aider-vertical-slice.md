# BENCHMARK — Vertical Slice Architecture

Implement the Task Manager REST API using the **Vertical Slice** pattern.

**Stack**: Java 21, Spring Boot 3.2, Maven  
**Storage**: in-memory (ConcurrentHashMap)

> Fix any failures and continue until all tests pass with ≥ 80% line coverage.

## Package Structure

```
src/main/java/com/benchmark/taskmanager/
├── TaskManagerApplication.java
├── task/                            ← everything task-related lives here
│   ├── TaskController.java          ← @RestController, @RequestMapping("/tasks")
│   ├── Task.java                    ← model
│   ├── CreateTaskRequest.java       ← @NotBlank on title only
│   ├── UpdateTaskRequest.java       ← title nullable (no @NotBlank)
│   ├── TaskService.java             ← business logic (single class or interface+impl)
│   └── InMemoryTaskRepository.java  ← ConcurrentHashMap storage
└── shared/
    └── exception/
        ├── TaskNotFoundException.java
        └── GlobalExceptionHandler.java ← @RestControllerAdvice

src/test/java/com/benchmark/taskmanager/
└── TaskManagerTest.java
```

### Vertical Slice Principle
All classes related to the "task" feature live inside `task/`.
No cross-feature imports. Shared utilities go in `shared/`.

---

## Data Model

```java
String id;          // UUID generated by server
String title;       // required, max 200 chars
String description; // optional, max 1000 chars
boolean completed;  // default false
String createdAt;   // ISO 8601 UTC
String updatedAt;   // ISO 8601 UTC, updated on every PUT
```

## Endpoints

| Method | Path         | Success | Error     |
|--------|--------------|---------|-----------|
| GET    | /tasks       | 200     | —         |
| POST   | /tasks       | 201     | 400       |
| GET    | /tasks/{id}  | 200     | 404       |
| PUT    | /tasks/{id}  | 200     | 400 / 404 |
| DELETE | /tasks/{id}  | 204     | 404       |

---

## Validation Rules

- `title` absent or blank → 400: `{"error": "title is required"}`
- `title` > 200 chars → 400: `{"error": "title must not exceed 200 characters"}`
- `description` > 1000 chars → 400: `{"error": "description must not exceed 1000 characters"}`
- ID not found → 404: `{"error": "Task not found"}`

**CRITICAL — ID path variable**: Use `@PathVariable String id` (NOT `@PathVariable UUID id`).
Call `UUID.fromString(id)` in try/catch — `IllegalArgumentException` → throw `TaskNotFoundException` (→ 404).

---

## PUT Partial Update

Only update fields present in the request. Always refresh `updatedAt`.
